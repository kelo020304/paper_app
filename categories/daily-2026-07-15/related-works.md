# 每日精选 2026-07-15 — Related Works

## 总览

WALA、Lumo-2、Xiaomi-Robotics-U0与新增的 See like a Robot 分别从四个互补层次推进具身学习：WALA将无动作视频中的观测变化转化为受真实动作锚定的潜在动作；Lumo-2用动作—世界动力学与语言语义共同塑造预测性控制表征；Xiaomi-Robotics-U0以统一生成模型扩展具身场景、编辑与未来视频数据；See like a Robot 则解决跨相机部署时观测坐标与动作坐标不一致的问题，将视觉几何直接表达在机器人动作坐标系中。

前三者主要回答“如何获得更丰富、更预测性的控制数据与表征”，而 See like a Robot 进一步回答“这些表征应以哪个空间参考系被策略理解”。因此，机器人中心点图既可作为VLA的几何输入，也可能成为WALA语义—几何变化建模或U0生成数据的统一度量接口；但它本身不替代潜在动作学习、未来动力学或世界生成。

## 从端到端 VLA 到具备空间与未来约束的控制

RT-1、RT-2奠定了由当前视觉观测和语言指令直接预测低层机器人动作的VLA范式；OpenVLA、Octo与π0、π0.5则通过大规模视觉—语言预训练、跨机器人数据或统一生成式建模提升泛化。这类模型接口直接、部署高效，但通常依赖动作标注，并默认策略能从相机图像中自行推断与动作相关的空间关系。

这一假设在相机视角变化时尤其脆弱。Universal Pose Pretraining、KYC、4D-VLA与SpatialVLA向仍以相机为中心的输入加入位姿、相机或空间线索；OC-VLA则将动作重参数化到相机坐标系。See like a Robot 采取不同路线：利用深度和标定将每个像素反投影为末端中心化的机器人坐标系 XYZ 点图，使观测几何与动作接口从输入端就共享参考系。它不要求策略从相机线索隐式学会坐标变换，也不以改变动作坐标系来适配视角。

该表示的关键设计在于保留 \(H \times W\) 图像网格。与 RoboUniView、Adapt3R 等构建共享三维场景表示的方法相比，机器人中心点图不追求完整场景重建，而是以图像形态承载度量 3D 坐标，从而复用预训练 VLA 的二维视觉路径，并可与 RGB 逐元素融合。GeoVLA、PointVLA、3D-CAVLA、DepthVLA、Any3D-VLA 和 PointAct 通常通过专用 3D 编码器、几何分支或点云交互注入空间信息；See like a Robot 的代价更低，仅需每步 RGB-D 反投影与坐标变换，但相应地依赖可靠的深度与相机—机器人标定。

动作表征是另一条正交轴线。OpenVLA 的离散分桶，以及 FAST、Faster、RDT-2 等高效或 VAE 式表征，主要处理动作 token 化、压缩与重建。Lumo-2进一步强调，控制质量取决于潜空间是否由视觉动力学和语言语义塑造成适合决策的几何结构：先进行动作—世界动力学预对齐，再以语义动作码本和多任务目标完成 V–L–A 对齐。WALA则沿用VLA作为部署时策略骨干，以潜在动作对齐和未来语义—几何动力学，让无动作视频参与控制学习。

因此，See like a Robot 与 Lumo-2、WALA 并非替代关系：前者规范化“策略看到的空间”，后两者分别规范化“策略内部动作表征”及“动作应导致的未来变化”。对于跨相机、跨工作站的控制，机器人中心点图可降低观测—动作坐标失配；对于缺少动作标注或需要长时预测的情形，仍需潜在动作与动力学约束补足。

## 世界模型作为训练期、推理期与数据期的控制支撑

LingBot-VA、DreamZero、Motus 和 GR00T N1 等工作从视频预测、视频生成或视频—动作联合建模出发，将控制与未来视觉演化关联；Fast-WAM强调世界建模可主要作为训练信号，而非测试时必须执行的显式规划。LDA-1B则在结构化DINO潜空间中联合扩展动力学、策略和视觉预测。

Lumo-2与这一路线共享“预测性动作表征”的目标，但不同于像素级视频生成世界模型：它从预训练VLM表征中学习面向控制的轻量潜在物理变化，并在动作生成前显式形成未来动力学表征。相较Lumo-1采用显式、粗到细的文本推理和规划，Lumo-2转向潜在推理，以降低 token 复杂度与推理延迟，并借助历史动作上下文消解任务阶段歧义。

WALA与Fast-WAM更接近“世界模型作为训练期控制约束”的范式：未来建模不增加测试时的显式想象负担。其约束直接施加于VLA产生的潜在动作，预测的是DINOv3语义特征增量与稠密深度增量，而非像素级视频或绝对未来观测；这种增量式语义—几何预测聚焦于动作导致的变化。

See like a Robot 不构成未来预测世界模型，但为这类方法补充了稳定的几何参考系。WALA的深度增量若在机器人中心坐标系中表达，可更直接对应末端相对物体和场景的变化；Lumo-2的潜在未来动力学则可从视角不变、度量化的空间输入中获益。与此同时，点图只解决输入几何对齐，无法单独处理接触、遮挡、动力学不确定性或长时未来预测。

Xiaomi-Robotics-U0则将世界模型的角色前移至数据构建。相较Dreamer、WAM、DreamZero或Qwen-RobotWorld等以机器人轨迹、未来预测、规划或策略学习为中心的具身世界模型，U0从EMU3.5出发，持续保留T2I与X2I共训，并把静态多视角合成、编辑式迁移和视频预测统一为离散自回归序列。它更适合作为产生或扩增训练分布的显式生成器，而非直接定义控制潜变量的动力学约束。

## 从无动作视频学习潜在动作

LAPA从观测转移中学习离散潜在动作，将无动作视频转化为动作样监督；Moto以运动token连接视频预训练与机器人控制；UniVLA在DINO特征空间学习任务中心的潜在动作，以抑制任务无关变化。UniT将潜在动作拓展到人类至人形机器人的迁移与世界建模，villa-X则关注潜在动作如何更有效整合进VLA预训练和控制流程。

WALA并非仅将潜在动作作为预训练token、运动先验或动作压缩接口。其潜在动作模型以多尺度、稀疏未来的语义特征增量和深度增量为预测目标；在策略学习阶段冻结编码器以提供稳定对齐目标，同时保留解码器作为可训练的潜在世界模型。真实机器人动作监督则持续约束潜变量的可执行性，从而在同一潜在动作空间内统一无动作视频提供的动力学结构与有标注示范提供的控制语义。

从这一视角看，Lumo-2与WALA是两种互补设计：Lumo-2侧重以预测性、语义化潜空间提升VLA本身的对齐与决策；WALA侧重以语义—几何变化建模，使无动作视频中的物理结构能可靠迁移到真实动作控制。See like a Robot 则将潜在动作学习前的观测变化显式放到动作相关坐标系，尤其适合消除“同一物理运动在不同相机中呈现为不同像素运动”的歧义。

## 从通用生成到具身可控观测合成

Stable Diffusion、FLUX Kontext、GPT-Image-2.0、Show-o、Janus、Janus-Pro、Chameleon、BAGEL与EMU3.5将图像生成、编辑或图文统一建模扩展到较强的语义理解和指令控制；Genie、Genie2、Sora、MovieGen、Wan与Cosmos进一步覆盖视频生成或世界模拟。这类基础生成模型擅长开放域视觉内容，却通常不直接约束机械臂姿态、跨相机几何、工作空间布局和接触交互。

可控编辑与多视角生成分别提供两类相关能力。InstructPix2Pix、AnyDoor、OmniGen、GPT-Image-2.0与FLUX Kontext关注参考或指令驱动的单图语义修改；MVDream、SyncDreamer、Wonder3D、Era3D、Zero123++、CAT3D和MVDiffusion++强调跨视角一致性，通常服务于对象中心的三维资产生成。前者缺乏多相机机器人场景的几何约束，后者通常不处理机械臂状态、操作视角和交互阶段。

Xiaomi-Robotics-U0将两条路线合并到具身观测域：以多视角深度和结构化文本控制RGB迁移，并将工作空间、背景、无关物、目标物与光照拆为可组合的编辑维度。其目标不是生成一般三维资产，而是生成在机器人几何、相机视角与操作状态上保持一致、同时可被针对性扰动的观测。

See like a Robot 与此形成清晰分工。ViP、Rovi-Aug、Mirage 和 AnyCamVLA通过新视角合成、数据扩增或测试时视图规范化应对相机变化；U0也可生成多视角或扰动后的训练观测。机器人中心点图则不依赖生成式视图合成，而通过已知标定和深度把真实或合成图像映射到共享机器人坐标系。前者扩大视角与场景分布，后者减少同一几何状态跨视角时的表征差异；二者可结合，但都受制于合成几何一致性、深度质量和标定精度。

## 具身视频作为合成数据引擎

Open X-Embodiment、RoboCasa、ManiSkill、BridgeData V2与DROID提供真实或仿真的机器人数据基础；RoboMIND/RoboCOIN、GenieSim、InternData-A1与AgiBotWorld则从具身视频、仿真或数据增强角度扩大规模。MovieGen、Wan、Cosmos、Genie和Qwen-RobotWorld表明，视频合成和未来状态预测可成为这种扩展的生成支撑。

Xiaomi-Robotics-U0将场景合成、深度条件迁移、子任务—子目标预测和多帧率视频生成纳入同一模型，形成“合成初始场景—按指令滚动生成视频—用于策略训练”的数据引擎路径。这比将图像编辑、仿真资产构建和短视频生成串成彼此独立的管线更一致，也便于按失败模式定向产生视觉扰动。

若将U0生成的数据用于跨视角VLA训练，See like a Robot 提供了一个自然的消费接口：在保证相机内外参和深度一致性的前提下，可把不同视角、背景或光照下的观测压到相同的机器人中心几何表示，再由WALA或Lumo-2学习控制相关的变化与未来约束。不过，U0的闭环可扩展性证据目前主要来自真实任务中的视觉扰动增强实验，尚不能等同于已验证的无限时长自主数据循环；同样，点图方法的跨视角稳健性也依赖深度估计与标定在真实部署中的稳定性。

总体而言，U0最稳妥的定位是高保真、可组合的具身数据扩增与世界合成模型；See like a Robot 是面向预训练VLA的显式机器人系几何对齐机制；WALA和Lumo-2则分别把扩展观测转化为可执行潜在动作与预测性决策表征。
