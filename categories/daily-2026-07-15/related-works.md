# 每日精选 2026-07-15 — Related Works（增量更新）

## 总览

WALA、Lumo-2、Xiaomi-Robotics-U0 与 See like a Robot 分别推进潜在动作、预测性表征、具身数据生成与机器人系几何对齐。LaWAM 则补上了 VLA 与世界模型之间的接口：预测可由动作专家在测试时使用的潜在视觉子目标。

新增的 S²-VLA 将这一脉络延伸到端到端自动驾驶，但其重点不是未来状态建模，而是解决 VLM 语义表征与连续空间控制之间的结构失配：语义流负责意图与场景含义，独立空间流在地图与目标等监督下保留几何细节并输出 waypoint。它说明“把语言模型接入控制”并不自动带来更好的空间决策；语义、几何和动力学应按各自的信息尺度分工。

整体上，近期工作围绕三个相互衔接的问题展开：以几何或潜在动作定义可执行变化，以特征空间世界模型预测未来，以及以流分离、层级、语义条件和稀疏推理让预测或表征服务长时域控制。关键不在于是否“有世界模型”或“使用 VLM”，而在于控制相关结构是否被保留、能否稳定预测，并能以低延迟接入规划或动作生成。

## 从端到端 VLA 到具备空间与未来约束的控制

RT-1、RT-2 奠定了由视觉观测和语言直接预测低层动作的 VLA 范式；OpenVLA、Octo 与 π0、π0.5 通过视觉—语言预训练、跨机器人数据或统一生成式建模提升泛化。Gemini Robotics、GR00T N1 等进一步扩展了指令、物体与任务组合的语义覆盖。这类模型接口直接，但通常依赖动作标注，并要求策略从图像中隐式推断空间关系、任务阶段与未来后果。

在自动驾驶中，TransFuser 以图像与 LiDAR 融合进行端到端模仿学习，UniAD、VAD 以统一多任务或矢量场景表示连接感知和规划，PARA-Drive、DRAMA、ARTEMIS 与 DiffusionDrive 则分别从并行化、状态空间模型、MoE 自回归或扩散生成改进轨迹规划。它们大多仍直接由感知表征生成规划结果，语言语义与可解释推理不是主要接口。

S²-VLA 在这一分支中采取不同的结构性分工：VLM 语义流先建模驾驶意图和场景语义，空间流则绕开语言头的离散 token 瓶颈，以显式空间特征配合地图、目标等监督细化连续 waypoint。因而，它不同于仅在端到端规划器上增加语言输入，也不同于将语义 token 直接当作几何控制状态。其现有实证主要是 NAVSIM 上纯 SFT 条件的比较，尚不足以证明该分离在真实闭环驾驶、分布外地图或长时域交互中的普适优势。

DriveLM、DriveGPT4 与 ReasonPlan 将视觉问答、链式推理或逐步决策引入驾驶，强化了高层语义解释，但通常仍属于开环顾问式输出，需要下游控制器执行。VLA-Adapter、SimLingo、Orion、CoT-VLA、AutoVLA、ReCogDrive 与 ImagiDrive 则从层级动作桥接、语言—动作对齐、视觉推理、自适应计算、认知 token 或想象增强等角度扩展 VLA。相较这些路线，S²-VLA 的核心不在于增加文本推理长度或更换生成式规划器，而在于显式划分“语义定意图、空间定轨迹”的表征责任。

FLARE、World Guidance、GR00T N1 与 Being-H0.7 等开始通过未来查询、特征对齐或条件建模引入时间结构；但未来动态往往仍压缩在隐式 token 或条件中，未必构成动作生成器可显式调用的动作条件。LaWAM 的定位正在于此：它保留 VLA 的语义骨干，同时预测空间结构化的潜在视觉子目标，并将其作为下游动作专家的测试时条件。因而，它不是以世界模型替代策略，而是把预测到的未来作为策略接口的一部分。

Universal Pose Pretraining、KYC、4D-VLA、SpatialVLA 和 OC-VLA 分别引入位姿、相机或坐标系线索。See like a Robot 利用深度和标定将像素反投影为末端中心化的机器人系 XYZ 点图，使观测几何与动作接口共享参考系。S²-VLA 同样强调空间通路不能完全从语义 token 中恢复，但其空间约束来自驾驶地图与目标表征，而非机器人本体坐标系的显式三维对齐。相较 RoboUniView、Adapt3R 等完整三维场景表示，以及 GeoVLA、PointVLA、3D-CAVLA、DepthVLA 等专用 3D 分支，See like a Robot 保留二维视觉路径并以 RGB—XYZ 融合降低实现代价，但依赖可靠深度与标定。

特征世界模型则为几何或策略接口增加未来约束。DINO-WM 将冻结 DINOv2 的局部 patch token 作为状态，学习动作条件特征转移，并直接以目标图像特征作为 MPC 代价；V-JEPA 2 先从互联网视频学习视频表征，再以较少机器人轨迹学习动作条件动力学。LaWAM 与二者共享“在特征空间预测未来”的取向，但其预测结果不作为 CEM 的目标或代价，而是直接条件化动作生成器；这以较少测试时搜索换取对动作专家质量和子目标可执行性的更强依赖。S²-VLA 与这些工作互补：前者首先解决当前观测中语义—几何表征的分工，后者进一步处理动作条件下未来状态的可预测性。

## 具身视频作为合成数据与鲁棒训练引擎

Open X-Embodiment、RoboCasa、ManiSkill、BridgeData V2、DROID 等提供真实或仿真数据基础；RoboMIND/RoboCOIN、GenieSim、InternData-A1 与 AgiBotWorld 从具身视频、仿真和数据增强扩大规模。U0 将场景合成、深度条件迁移、子任务—子目标预测和多帧率视频生成纳入同一模型，形成“合成初始场景—按指令滚动生成视频—训练策略”的数据引擎。

这些世界模型提供了数据扩增之外的闭环利用方式：HWM 将离线动力学转为跨尺度规划器；ThinkJEPA 用 VLM 语义扩展潜动力学的有效时间感受野；不变视觉 JEPA 降低视觉域变化干扰；Sparse Imagination 使密集特征规划可按预算伸缩；RLA-WM 尝试在离线视频模型中训练策略；LaWAM 则将预测未来转化为 VLA 的低延迟动作条件。S²-VLA 不属于这一生成或预测式世界模型分支，但提供了其有效接入控制前的必要提醒：若语义表征在输入端已压缩关键空间关系，再强的后续预测模块也难以完全恢复精细轨迹所需几何。

因此，U0 解决“如何扩展并重组看见的数据”，See like a Robot 解决“如何让看到的几何与动作对齐”，S²-VLA 解决“如何避免语义推理吞没连续空间控制”，WALA、Lumo-2 与潜在动作世界模型解决“如何把观测变化转为可预测、可执行的控制变量”，DINO-WM、V-JEPA 2、DINO-world、LeWM、RLA-WM 与 LaWAM 解决“在何种潜空间建模未来、又如何将未来接入控制”，HWM、ThinkJEPA、不变视觉 JEPA 与 Sparse Imagination 则解决跨时间、语义、视觉域和计算预算下的可用性。它们可以组合，但视觉一致性、空间对齐、特征可预测性和低延迟接口都不能替代几何、接触与反事实动力学的真实性。
