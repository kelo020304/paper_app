# 每日精选 2026-07-15 — Related Works（增量更新）

## 总览

近期工作正沿着一条相互依赖的链路推进具身智能：以几何或潜在动作定义可执行变化，以特征世界模型预测未来，再通过流分离、层级接口与稀疏推理将预测结果接入控制。S²-VLA 强调语义表征与连续空间决策应分工；LaWAM 则预测可供动作专家在测试时调用的潜在视觉子目标。

新增的 Open-AoE 补足了这条链路的数据与工程基础。它不提出新的 VLA 或世界模型算法，而是将低成本第一视角采集、MANO 手部表示、相机轨迹、原子动作及处理工具整合为开放接口，使人类第一视角视频能更稳定地进入重建、重定向、模仿学习和世界模型训练流程。由此，问题不只在于“能否预测未来”，也在于训练数据是否保留了手—物运动、相机自运动和动作边界等控制相关结构。

## 从端到端 VLA 到具备空间与未来约束的控制

RT-1、RT-2 奠定了由视觉观测和语言直接预测低层动作的 VLA 范式；OpenVLA、Octo 与 π0、π0.5 通过视觉—语言预训练、跨机器人数据或统一生成式建模提升泛化。Gemini Robotics、GR00T N1 等进一步扩展指令、物体与任务组合的语义覆盖。这类模型接口直接，但通常依赖动作标注，并要求策略从图像中隐式推断空间关系、任务阶段与未来后果。

在自动驾驶中，TransFuser 以图像与 LiDAR 融合进行端到端模仿学习，UniAD、VAD 以统一多任务或矢量场景表示连接感知和规划，PARA-Drive、DRAMA、ARTEMIS 与 DiffusionDrive 则分别从并行化、状态空间模型、MoE 自回归或扩散生成改进轨迹规划。它们多数仍直接由感知表征生成规划结果，语言语义与可解释推理不是主要接口。

S²-VLA 的关键是结构分工：VLM 语义流建模驾驶意图和场景含义，独立空间流则以地图、目标等监督保留连续几何细节并输出 waypoint。它因此不同于仅为端到端规划器增加语言输入，也不同于将语义 token 直接当作几何控制状态。其现有结果主要基于 NAVSIM 的纯 SFT 比较，真实闭环、分布外地图和长时域交互中的优势仍有待验证。

DriveLM、DriveGPT4 与 ReasonPlan 将视觉问答、链式推理或逐步决策引入驾驶，强化高层语义解释，但通常仍需要下游控制器执行。VLA-Adapter、SimLingo、Orion、CoT-VLA、AutoVLA、ReCogDrive 与 ImagiDrive 从层级动作桥接、语言—动作对齐、视觉推理、自适应计算、认知 token 或想象增强等方向扩展 VLA。相较这些路线，S²-VLA 的贡献不在于增加推理文本或更换生成器，而在于显式划分“语义定意图、空间定轨迹”的表征责任。

FLARE、World Guidance、GR00T N1 与 Being-H0.7 开始通过未来查询、特征对齐或条件建模引入时间结构，但未来动态常仍压缩在隐式 token 或条件中。LaWAM 则保留 VLA 的语义骨干，预测空间结构化的潜在视觉子目标，并将其作为下游动作专家的测试时条件；它不是以世界模型替代策略，而是把预测到的未来变成策略接口的一部分。

Universal Pose Pretraining、KYC、4D-VLA、SpatialVLA 和 OC-VLA 引入位姿、相机或坐标系线索。See like a Robot 利用深度和标定，将像素反投影为末端中心化的机器人系 XYZ 点图，使观测几何与动作接口共享参考系。S²-VLA 同样指出空间通路不能完全从语义 token 中恢复，但其约束来自驾驶地图与目标，而非机器人本体坐标系的显式三维对齐。

特征世界模型进一步提供未来约束。DINO-WM 将冻结 DINOv2 的局部 patch token 作为状态，学习动作条件特征转移，并以目标图像特征构造 MPC 代价；V-JEPA 2 先从互联网视频学习视频表征，再用较少机器人轨迹学习动作条件动力学。LaWAM 与二者共享在特征空间预测未来的取向，但其预测结果直接条件化动作生成器，而非用于 CEM 搜索；这降低测试时规划负担，也更依赖动作专家质量和子目标可执行性。S²-VLA 与这些方法互补：前者先保证当前观测中的语义—几何分工，后者处理动作条件下未来状态的可预测性。

## 具身视频作为数据、几何监督与训练接口

Open X-Embodiment、RoboCasa、ManiSkill、BridgeData V2、DROID 等构成真实与仿真机器人数据基础；RoboMIND/RoboCOIN、GenieSim、InternData-A1 与 AgiBotWorld 通过具身视频、仿真和数据增强扩大规模。U0 进一步将场景合成、深度条件迁移、子任务—子目标预测和多帧率视频生成纳入同一模型，形成“合成初始场景—按指令滚动生成视频—训练策略”的数据引擎。

Open-AoE 位于这一脉络的人类第一视角分支。EPIC-KITCHENS-100 与 Ego4D 提供大规模日常活动和语言覆盖，但缺少直接的手姿态或相机轨迹；HOI4D、HOT3D、EgoMimic、EgoDex 与 EgoLive 则逐步增强 RGB-D、多视角、手—物运动、相机运动及桌面操作标注。OpenEgo 的重点是跨来源统一，EgoScale 则探索超大规模野外视频。相比之下，Open-AoE 不以最大数据量或专用传感器精度为目标，而以消费级手机、自然环境采集和端到端开放工具链形成“低成本连续采集—结构化处理—模型接入”的闭环。

这一定位也决定了其几何取舍：Open-AoE 同步发布 MANO、相机轨迹、有效性掩码和原子动作边界，足以支持按手、物与自运动分解训练；但其几何主要来自重建，不能等同于多视角动捕或专用传感器的真值精度。它适合作为规模化、开放式训练基础设施，而高精度接触、遮挡和细粒度操作评估仍需要与 HOT3D、HOI4D 等更强传感数据互补。

从人类视频到机器人学习的接口上，HaWoR、EgoInfinity、EgoAERO 与 EgoEngine 分别处理世界坐标手运动恢复、度量化 4D 手—物轨迹、接触一致轨迹和机器人化观测—动作生成；Phantom、SPIDER 等则关注视觉迁移或物理约束重定向。Open-AoE 的贡献并非替代这些重建或 IK 后端，而是将它们所需的同步手部表示、相机运动、有效性信息和动作片段组织为共同输入，降低不同后端接入训练流程的摩擦。

在学习范式上，Being-H0、EgoVLA、VITRA 将人类视频动作表征引入 VLA 预训练；DreamZero、LaWAM 探索世界动作模型；iVideoGPT、Ctrl-World、DreamDojo 探索交互式或可控世界模型。Open-AoE 同时服务这些异构方向：它提供不同粒度的动作转换接口，并显式保留相机运动，使手—物操作与第一视角自运动既可联合建模，也可按任务拆分。其价值因此在于扩大可用训练信号，而不是宣称由视频重建本身解决机器人动力学或接触真实性。

HWM、ThinkJEPA、不变视觉 JEPA 与 Sparse Imagination 则从跨尺度规划、语义扩展、不变视觉表征和计算预算控制提升世界模型的可用性。综合来看，U0 解决“如何扩展和重组看见的数据”，Open-AoE 解决“如何将自然第一视角视频整理为可接入的结构化训练信号”，See like a Robot 解决“如何让视觉几何与机器人动作对齐”，S²-VLA 解决“如何避免语义推理吞没连续空间控制”，而 DINO-WM、V-JEPA 2、LaWAM 等解决“如何预测未来并将其接入控制”。这些模块可以组合，但数据规模、视觉一致性与低延迟接口都不能替代可靠的几何、接触和反事实动力学。
