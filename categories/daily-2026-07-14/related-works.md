# 每日精选 2026-07-14 — Related Works

## 总览

端到端自动驾驶正从“感知特征到自车规划”的统一学习，扩展为同时建模语言语义、场景状态与多智能体未来交互。WCog-VLA沿此脉络提出双层世界认知：以视觉语言模型提供高层语义与推理，以显式的三维智能体状态和联合轨迹生成支撑前瞻规划。

## 端到端规划与生成式驾驶

早期端到端方法如 TransFuser 将多传感器观测直接映射为驾驶决策；UniAD 与 VAD 则以统一的稠密 BEV 表示衔接感知、预测和规划。SparseAD进一步以稀疏查询降低该类统一架构的计算开销。

为应对未来行为的多模态性，GenAD、DiffusionDrive 和 VADv2 等方法将规划转向生成式或概率式轨迹建模。这些工作主要聚焦于提升自车未来轨迹分布的表达能力。相比之下，WCog-VLA不仅生成自车轨迹，还联合预测周边智能体的当前与未来状态，并通过 ADDT 建模自车与其他交通参与者的联合轨迹；其重点因而从“更好地采样自车未来”延伸为“基于显式交互世界模型进行规划”。

## VLM/VLA 驾驶与连续动作建模

视觉语言模型最初更多承担驾驶场景的语义理解和解释角色，如 DriveGPT4、DriveLM。后续 SENNA、EMMA、SimLingo 和 DriveMoE 等工作开始将多模态输入直接关联至驾驶动作，推动 VLM 从描述器走向策略模型。

在动作表示上，OmniReason、AdaThinkDrive 使用文本推理生成决策；AutoVLA 采用自回归动作 token；ORION 借助 VAE、ReCogDrive 借助扩散式规划，以缓解离散语言表征与连续控制之间的鸿沟。WCog-VLA同样以视觉语言模型为高层认知基础，但不将其局限于文本推理或自车动作解码：其 agent token 受到三维感知和未来运动监督，并与联合多智能体扩散轨迹生成相连接，从而把语义推理落到可验证的物理状态与交互未来上。

## 世界认知与多智能体前瞻

围绕 VLA 的世界认知，DrivePI 强调空间感知，SGDrive以 scene–agent–goal 结构组织驾驶认知，SparseOccVLA等工作则加强占据或空间表示。另一条路线如 UniDrive-WM 与 DriveVLA-W0，通过未来图像生成赋予模型前瞻能力。

WCog-VLA将这些方向结合为“语义预测 + 轨迹生成”的双层设计：高层通过 VLM 与 Game-CoT 进行面向交互的推理，低层以联合多智能体轨迹生成刻画物理演化。相较于侧重语义结构、空间表示或感知式未来生成的路线，其核心差异在于显式优化自车—周边智能体的联合运动关系，并以 Stackelberg 式博弈视角服务规划决策。
