VistaVLA: Geometry- and Semantic-Aware 3D Gaussian-Grounded VLA for Robotic Manipulation

价值 8/10 论文将“语义对齐的三维场景表示”与“固定长度的策略接口”同时落地，且给出了真实机器人、标准模拟和空间 OOD 的对照与消融。其最有价值的证据是：在不依赖大规模动作预训练的设置下，标准 LIBERO 和 Pro-Swap 均优于所列无预训练基线。不过，实测场景与本体范围较窄、试验次数有限，且尚未证明对动态或弱标定部署的泛化，故评为高价值但非决定性证据。

VistaVLA 将多视角视觉语义特征蒸馏到带几何锚点的 3D Gaussian primitive 中，再以 Merge-then-Query（MtQ）压缩为 64 个策略上下文 token，以提升 VLA 机器人操作的空间理解与空间分布外鲁棒性。

研究问题

以图像为主要输入的 VLA 虽能把语言和视觉映射到动作，但缺少显式、场景级的三维表示；而加入深度、点云或坐标的既有 3D VLA 多偏向低层几何，缺乏与视觉-语言特征对齐的三维语义锚定。直接把约 10^5 个稠密 Gaussian primitive 输入策略网络又难以满足实时控制的计算约束。

核心贡献

  • 提出两阶段 VistaVLA：第一阶段以 RGB-D 渲染和教师特征监督学习语义 Gaussian 场；第二阶段把其压缩并注入 VLA-Adapter 式策略。
  • 把 SigLIP2 与 DINOv2-Large 的拼接特征经自编码器从 2176 维压缩到 128 维，并作为每个 Gaussian primitive 的语义特征；训练时通过多视角特征渲染监督，使 token 同时保持 2D 教师空间对齐与 3D 几何/可见性约束。
  • 提出 MtQ：先将每视图 token 网格进行 stride-4 平均池化，再基于 Morton code 的空间排序和语义余弦相似度进行无参数合并，压至 1000 个 token；随后用两层 Transformer decoder 的 64 个可学习查询 token 汇总为策略可消费的 3D 上下文。
  • 在真实机器人、LIBERO 标准套件、LIBERO-Pro-Swap 空间 OOD 及消融中报告性能提升，并检验相机视角数、压缩策略和查询 token 数的影响。

方法原理

Stage I：每个 Gaussian 表示为中心、协方差、不透明度、颜色及 128 维语义向量。先以 RGB 和深度重建建立稳定几何，再保持 RGB-D 损失并加入特征渲染的 L2 监督。训练使用两个标定 RGB-D 上下文视图、渲染第三个新视角以促进跨视图一致性；推理时依据实时 RGB 图像和标定相机位姿在线构建 Gaussian 场，论文称不需要深度输入或预建持久地图。Stage II：MtQ 以空间中心作为分区元数据、语义特征作为匹配依据；仅合并相似度最高的 50% 非锚点 token，以避免强制合并语义不同的 token，并以聚合大小加权更新特征和中心。64 个汇总 token 经投影后与视觉 token、语言 token、动作查询 token 拼接，采用因果序列建模和 L1 动作回归预测未来 K 步连续动作；真实机器人每次执行前 4 个预测动作后重规划。

实验与证据

真实环境：在 Agile-Piper 平台的 7 个任务（PlaceSponge、StackBoxes、StackBowls、PlacePlate、OrganizeSponge、ThrowTrash、Wand&Cup）上，每种方法每任务 10 次试验，使用相同任务示范与匹配训练预算；基座为 VLA-Adapter-0.5B，并比较 VLA-Adapter+Depth、SmolVLA-0.5B、π0.5-3B。论文摘要称 VistaVLA 的七任务平均成功率较基线高 22.8 个百分点；在挑战性 OOD 任务上较 VLA-Adapter 高 30.0 个百分点。空间扰动中，PlaceSponge 深度/高度变化为 9/10，VLA-Adapter 和 +Depth 为 6/10、π0.5 为 7/10；OrganizeSponge 位置变化为 3/10，其他列出的基线均为 0/10。模拟环境：LIBERO 四套件平均成功率为 96.05%，高于无大规模机器人动作预训练的 VLA-Adapter 的 94.1%；LIBERO-Pro-Swap 平均为 12.2%，相对该基线的 1.7% 提升 10.5 个百分点。消融显示：3 相机的普通基线使用 768 个 2D token，而 VistaVLA 使用 320 个 token 且在所列四项任务更好；完整 MtQ 在三项真实任务为 9/10、7/10、8/10，优于 FPS/熵采样/线性汇总变体；查询 token 取 64 时在 StackBowls 为 8/10，优于测试的 16、32、128、256。

Related Works 表述

VLA 主线中，Brohan 等的 RT-1、Zitkovich 等的 RT-2、Kim 等的 OpenVLA、Octo 团队的 Octo 及 Wang 等的 VLA-Adapter 将图像与语言直接映射到动作，但本文将其概括为主要依赖二维观测，未显式表征场景级三维关系。为补足空间信息，Zhen 等的 3D-VLA、Li 等的 PointVLA、Qu 等的 SpatialVLA、Bhat 等的 3D-CAVLA，以及深度增强类方法，向策略加入点云、三维坐标或深度；VistaVLA 相对这类方法的差异是：不把几何信号作为孤立的额外输入，而用多视角一致的语义 Gaussian 将语言对齐视觉语义绑定到三维 primitive，再以紧凑 token 接入 VLA。另一方向是 Li 等的 Spatial Forcing、Zhang 等的 InSpire 等通过空间感知预训练或表征对齐隐式改善推理；VistaVLA 则保留预训练 VLM/VLA 所需的二维特征对齐，并显式构建三维、场景级接口，作者认为可减少强对齐对原生特征空间的扰动风险。3D Gaussian Splatting 方面，Kerbl 等提出 3DGS；Qin 等的 LangSplat、Zhou 等的 Feature 3DGS 将语言或基础模型特征附着于 Gaussian；Ji 等的 GraspSplats、Zheng 等的 GaussianGrasper、Lu 等的 ManiGaussian/GWM 则将 Gaussian 用于抓取、操作世界模型或 SLAM。VistaVLA 的定位差异在于把语义 Gaussian 明确作为面向策略的 VLA 控制上下文，而非主要服务于新视角合成、场景理解、地图或单独抓取模块。效率方面，Bolya 等的 ToMe、Rao 等的 DynamicViT、Ryoo 等的 TokenLearner 侧重二维视觉 token 的合并、裁剪或选择，Alayrac 等的 Flamingo 和 Li 等的 BLIP-2 以查询瓶颈连接视觉与语言；LightGaussian、Compact 3D Gaussian Representation、ContextGS 则压缩 Gaussian 的存储/渲染。VistaVLA 的 MtQ 区别是先以 Morton 引导的空间分区结合语义相似度合并三维语义 token，再用查询解码器形成固定长度、供动作预测使用的三维摘要，目标是保留操作相关的几何、语义与多视角一致性。
